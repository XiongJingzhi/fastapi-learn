## 📝 自我评估问题

### 问题 1：基础理解（必答）

**用你自己的话，解释"并发"和"并行"的区别。**

提示：可以用生活中的例子类比。

<details>
<summary>查看参考答案</summary>

**并发**：多个任务在同一个时间段内轮流执行，看起来像同时进行，实际上是交替进行。

**并行**：多个任务在同一时刻真正同时执行。

**生活例子**：
- 并发：一个人同时做饭和看电视（实际是轮流做）
- 并行：两个人，一个做饭，一个看电视（真正同时做）
</details>

---

### 问题 2：场景判断

**判断以下场景是并发还是并行，并说明原因。**

1. 单核 CPU 上同时运行浏览器和音乐播放器
2. 多核 CPU 上，一个核心计算，另一个核心处理网络请求
3. 你一个人在手机上边听音乐边刷社交媒体
4. 双核 CPU 的电脑上同时运行两个独立程序

<details>
<summary>查看答案</summary>

1. **并发** - 单核 CPU 只能轮流执行任务
2. **并行** - 多核 CPU 可以真正同时执行
3. **并发** - 一个人只能交替做两件事
4. **并行** - 两个核心独立工作，真正同时进行
</details>

---

### 问题 3：Python 特性

**为什么说"Python 的多线程是并发，不是并行"？**

<details>
<summary>查看答案</summary>

因为 Python 有 **GIL（全局解释器锁）**：
- GIL 规定同一时刻只能有一个线程执行 Python 字节码
- 多线程只能轮流执行（并发）
- 无法真正同时执行（不是并行）
- 要实现并行，必须用多进程（multiprocessing）
</details>

---

### 问题 4：实际应用

**以下场景应该用并发还是并行？为什么？**

A. 爬虫程序，需要抓取 1000 个网页
B. 图片处理程序，需要给 1000 张照片添加滤镜
C. Web 服务器，处理用户请求
D. 科学计算，进行大量数值运算

<details>
<summary>查看答案</summary>

A. **并发** - IO 密集型，大部分时间等待网络响应，适合多线程或异步

B. **并行** - CPU 密集型，需要大量计算，应该用多进程充分利用多核

C. **并发** - IO 密集型，等待数据库、外部 API 等，FastAPI 用异步处理

D. **并行** - CPU 密集型，需要持续计算，应该用多进程
</details>

### 问题 5：费曼测试（挑战）

**想象你在向一个完全不懂技术的朋友解释。请用不超过 3 句话，解释清楚并发和并行的区别。**

<details>
<summary>参考答案</summary>

1. 并发是一个人轮流做多件事，看起来同时做，实际是交替进行。
2. 并行是多个人同时做各自的事，真正同时进行。
3. 一个人边吃饭边看电视是并发，两个人各自吃饭是并行。
</details>
